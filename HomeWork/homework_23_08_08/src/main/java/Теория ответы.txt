Что является объектом в java?

Объект в Java представляет собой экземпляр класса.
Класс определяет структуру и поведение объекта,
а экземпляр класса — это конкретное воплощение этой структуры,
которое может хранить данные (поля) и выполнять определенные действия (методы).
В Java каждый объект является экземпляром какого-либо класса.
----------------------------------------------------------------------------------------------------
Как определяется какой метод будет вызван в случае передачи туда примитива? объекта?

В случае если несколько методов с одним и тем же именем, но различными параметрами
(типами и/или количеством), то
когда метод вызывается, компилятор выбирает наиболее подходящий метод на основе переданных аргументов.
Если есть методы, соответствующие типу переданных аргументов, то компилятор выберет наиболее
точное совпадение.
Если есть общий интерфейс или базовый класс, методы которого переопределены в подклассах,
то при вызове метода на объекте подкласса будет вызван метод из этого подкласса.
В случае передачи примитивных типов данных и объектов, процесс выбора метода будет зависеть
от типов аргументов, и компилятор выберет наиболее подходящий метод на основе сигнатуры метода
(типов и количества аргументов).
----------------------------------------------------------------------------------------------------
Что такое autoboxing и unboxing?

Autoboxing и unboxing - это механизмы в Java, которые автоматически преобразуют между
примитивными типами данных и их соответствующими обёртками (wrapper classes).
Эти механизмы позволяют работать с примитивами как с объектами и наоборот,
без необходимости явного преобразования.
----------------------------------------------------------------------------------------------------
Почему не стоить сравнивать через ==

В Java оператор == сравнивает примитивные типы данных на равенство значений,
а для объектов он сравнивает ссылки на объекты, а не сами объекты по их содержимому.
Это означает, что два объекта могут содержать одинаковые данные, но при этом сравнение
obj1 == obj2 вернет false, если obj1 и obj2 - это разные ссылки на объекты,
даже если они хранят одинаковые значения.
Для сравнения содержимого объектов (а не ссылок) следует использовать методы equals()
или другие специальные методы для сравнения, которые могут быть реализованы в собственных классах.
----------------------------------------------------------------------------------------------------
Сколько весит в памяти ссылка?

Ссылка на объект в Java обычно занимает 4 байта на 32-битной JVM и 8 байт на 64-битной JVM.
Это так называемый "размер указателя" или "размер ссылки". Фактический объем памяти,
занимаемый ссылкой, может изменяться в зависимости от реализации JVM и настроек сборщика мусора.
----------------------------------------------------------------------------------------------------
Почему объекты называются ссылочными типами данных?

Объекты в Java называются ссылочными типами данных потому, что переменные, объявленные как
ссылочные типы (классы или интерфейсы), хранят не сами объекты, а лишь ссылки (адреса) на эти
объекты в памяти. Примитивные типы данных (int, char, double и т.д.) хранят свои значения
непосредственно в переменной, в то время как объекты сами по себе представляют сложные
структуры данных и хранятся в динамической области памяти, недоступной напрямую по адресу.
